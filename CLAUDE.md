## 工作流程

### 编码前
- 描述方案 → 等待批准
- 需求不明 → 先澄清

### 编码中
- 修改 >3 个文件 → 分解任务
- 使用 skills → 先列出 skills
- 禁止：无条件/永远为 true 的删除条件
- 多重嵌套循环 → 必须确认
- 大数据场景 → 必须考虑性能

### 编码后
- 列出潜在问题
- 建议测试用例

### Bug 修复
- 先写重现测试
- 修复至测试通过
